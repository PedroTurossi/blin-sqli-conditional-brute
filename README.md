# Brute-force de Blind-SQLi com respostas condicionais, error-based e time-based
Desenvolvido durante testes nos laboratórios da Port Swigger de SQLi (SQL Injection), os arquivos acima são maneiras automatizadas de injeção SQL através de Cookies do cabeçalho HTTP. Em todos os casos, tem um banco de dados diferente (MySQL, Postgre...) e uma situação diferente (time-based, respostas condicionais ou error-based).
Logo, foi possivel implementar uma lógica que pegava pedaço por pedaço da informação e conseguir tudo o que era necessário para concluir os laboratórios automaticamente. Em todos os casos, era necessário obter a senha do administrador através de diferentes técnicas de SQLi.
